Manage API keys for programmatic access to the Chanl API.

API keys provide a way to authenticate API requests without requiring user interaction. They are scoped to specific workspaces and can have limited permissions.

## List API Keys

<RequestExample>

```bash cURL
curl -X GET "https://api.chanl.ai/api/v1/auth/keys" \
  -H "Authorization: Bearer <access_token>"
```

```javascript JavaScript
const response = await fetch('https://api.chanl.ai/api/v1/auth/keys', {
  headers: {
    'Authorization': 'Bearer ' + accessToken
  }
});

const { data } = await response.json();
console.log('API Keys:', data.keys);
```

```python Python
import requests

response = requests.get(
    'https://api.chanl.ai/api/v1/auth/keys',
    headers={'Authorization': f'Bearer {access_token}'}
)

keys = response.json()['data']['keys']
```

## Create API Key

<RequestExample>

```bash cURL
curl -X POST "https://api.chanl.ai/api/v1/auth/keys" \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Analytics Dashboard",
    "permissions": ["read"],
    "expires_at": "2024-12-31T23:59:59Z"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.chanl.ai/api/v1/auth/keys', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer ' + accessToken,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Analytics Dashboard',
    permissions: ['read'],
    expires_at: '2024-12-31T23:59:59Z'
  })
});

const { data } = await response.json();
// Store the API key securely - it won't be shown again
console.log('New API Key:', data.api_key);
```

```python Python
import requests

response = requests.post(
    'https://api.chanl.ai/api/v1/auth/keys',
    headers={'Authorization': f'Bearer {access_token}'},
    json={
        'name': 'Analytics Dashboard',
        'permissions': ['read'],
        'expires_at': '2024-12-31T23:59:59Z'
    }
)

api_key = response.json()['data']['api_key']
# Store securely - won't be shown again
```

</RequestExample>

## Delete API Key

<RequestExample>

```bash cURL
curl -X DELETE "https://api.chanl.ai/api/v1/auth/keys/key_123" \
  -H "Authorization: Bearer <access_token>"
```

```javascript JavaScript
const response = await fetch('https://api.chanl.ai/api/v1/auth/keys/key_123', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer ' + accessToken
  }
});

const { data } = await response.json();
console.log('Key deleted:', data.message);
```

```python Python
import requests

response = requests.delete(
    'https://api.chanl.ai/api/v1/auth/keys/key_123',
    headers={'Authorization': f'Bearer {access_token}'}
)

message = response.json()['data']['message']
```

</RequestExample>

<ResponseExample>

```json Delete Key Response
{
  "success": true,
  "data": {
    "message": "API key deleted successfully"
  }
}
```

```json Error Response

  "error": {
    "code": "KEY_NOT_FOUND",
    "message": "API key not found"
  }
}
```

</ResponseExample>

## API Key Permissions

<ParamField path="permissions" type="array">
  Array of permission strings
</ParamField>

Available permissions:
- `read` - Read access to all workspace resources
- `write` - Create and update resources
- `delete` - Delete resources
- `admin` - Full administrative access

## Security Best Practices

<Warning>
API keys are only shown once during creation. Store them securely and never expose them in client-side code.
</Warning>

- Use the principle of least privilege - grant only necessary permissions
- Set expiration dates for temporary integrations
- Regularly rotate API keys
- Monitor key usage through the dashboard
- Revoke compromised keys immediately
- Use environment variables to store keys in applications
